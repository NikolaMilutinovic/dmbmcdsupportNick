SNOW

A basic guide on Snow cases

Snow cases are created by L2 DMB Support team either for the Deployment team to install the already ordered screens, or the L2 DMB Support team for further troubleshooting and investigation.
The first visit can be successful, without issues, where a pick-up is requested if possible, and Snow and Atos tickets are closed, or it can be completed with issues, where a re-visit is scheduled, and Snow case is updated.
Once an Atos ticket gets assigned to VDR:NCR-DMB, and it contains the notes that Displays have been ordered and approved through the Samsung portal, a SNOW Case is created with the following template:
Store Number: (Store Number)
Account: (MCN number of the store, gathered from the Master Restaurant List)
Primary Contact: (After selecting Account, press the magnifying glass, choose a contact)
Assignment Group: 31 CCC Deployment
Initiative: (Atos INC number)
Overall Case State: Starting
Program: DMB Move/Add/Change (MAC)
Planned Install Type: NCR Professional Install
Planned Install State: Deliverables Pending
Planned Install Date: (Last day of the year)
Classification: Project
Deployment Type: Move/Add/Change
Template: BF / Non BF (If the instalation is covered, BF, if not, Non BF)

If the SNOW ticket is created for a troubleshooting/investigation process by L2 DMB Support team, the following field in the tempate should be changed. Program: DMB MAC Pre Order
